This helps you to set up an organization-wide role hierarchy and share rules between them. A security role defines how different users, can access different types of records.

### Adding Roles

To add a new Role:


*   Navigate to your**Profile Icon**on the top right corner.

*   Go to**Set Up**.

*   Head over to**Users & Security***   Click on the**Security Roles**![](./Managing Security Roles in Salesmate – Salesmate_files/mceclip1.png)


*   Here, you will be presented with the List of Roles.

![](./Managing Security Roles in Salesmate – Salesmate_files/mceclip2.png)


***Hover**over a**Role**(under which you want to add a Role)

*   Click on (**+**)**Add Role**![](./Managing Security Roles in Salesmate – Salesmate_files/mceclip3.png)


*   Enter the Role**Name***   Choose the**Reporting****role***   Describe the**Role***   Set the**Data sharing with peers**(Selection of this option allows data to be shared with users on the same Role)

![](./Managing Security Roles in Salesmate – Salesmate_files/mceclip4.png)


### Assigning Roles

Since the Roles of users evolve within the organization. It may require that you may need to update the**Roles**and**Profile****Permissions**of users. Here are steps to update Roles and Profile Permissions :


*   Navigate to your**Profile Icon**on the top right corner.

*   Go to**Set Up**.

*   Head over to**Users****& Security**.

*   Click on**Users**![](./Managing Security Roles in Salesmate – Salesmate_files/mceclip0.png)


*   Here you get the list of all**Users**.

*   Hover over to the**User Name,****Actions**button will appear next to the name.

*   Click on the**Actions**Button and select**Edit**![](./Managing Security Roles in Salesmate – Salesmate_files/mceclip5.png)


*   Go to**Roles**Field to update the information

*   Hit**Update**to save the changes.
